# AutomationExercise-qa-portfolio 

A personal QA portfolio project demonstrating manual testing skills
on a real-world e-commerce application.

## Application Under Test
- **Website:** [Automation Exercise](https://automationexercise.com)
- **Type:** E-commerce web application
- **Testing Environment:** Chrome browser, Windows 11

## Tools Used
- Jira - test case and bug management
- Chrome DevTools - UI inspection & debugging

## Project Structure
```
├── test-plan/         # Test plan document
├── test-cases/        # Test case summary by module
├── bug-reports/       # Bug report summary
├── api-testing/       # Coming soon
└── sql-validation/    # Coming soon
```

## Test Coverage
Modules covered:
- User Registration — 5 test cases
- Login / Logout — 5 test cases
- Search Functionality — 4 test cases
- Add to Cart — 4 test cases
- Checkout Flow — 4 test cases
- Total: 22 test cases

## Bugs Found
- Total bugs reported: 5
- BUG001 — Invalid email format accepted during registration
- BUG002 — Name field accepts spaces only
- BUG003 — Session persists after navigating back without logout
- BUG004 — Search does not trigger on pressing Enter key
- BUG005 — No option to decrease individual item quantity in cart
