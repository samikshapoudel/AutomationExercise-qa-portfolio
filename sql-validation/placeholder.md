# SQL Data Validation - Automation Exercise

## Status: In Progress

## Purpose
Using T-SQL queries to validate data integrity and accuracy behind the application. This section demonstrates SQL-based quality checks that a QA engineer would run to verify the database reflects what the UI shows.

## Planned Validation Queries
- Verify no duplicate user accounts exist for same email
- Verify user account is created after successful registration
- Verify user account is deleted after delete account API call
- Verify product list data is not null or empty
- Verify order is created after successful checkout
- Verify no null values in required fields like name and email

## Tools
- T-SQL
- SQL Server Management Studio (SSMS)

## Note
SQL validation queries will be added to this folder once database access is configured.
